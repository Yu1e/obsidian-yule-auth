# yule-auth v3.0

Плагин для Obsidian: отслеживание и визуализация авторства текста — своего, чужого или ИИ.

---

## Что нового в v3.0

### 1. Якорная привязка фрагментов (anchor-based tracking)

Авторство теперь хранится **не как позиция в файле**, а как якорный текст — первые и последние 40 символов фрагмента. При поиске якоря допускается сдвиг до 200 символов от сохранённой позиции.

- **Небольшие правки и добавления** → авторство сохраняется.
- **Серьёзное переписывание** (якорный текст исчез) → авторство снимается автоматически.
- Хранится: `startAnchor`, `endAnchor`, `savedFrom`, `savedTo`.

### 2. Авторство всей заметки (noteClass)

Заметке можно присвоить класс авторства целиком. Внутри неё отдельные фрагменты могут быть **исключениями** с другим классом — они помечаются пунктирной рамкой.

**Пример:**
```
ЧУЖОЙ ТЕКСТ ЧУЖОЙ ТЕКСТ ЧУЖОЙ ТЕКСТ [мой комментарий] ЧУЖОЙ ТЕКСТ ЧУЖОЙ ТЕКСТ
```
Вся заметка — `other`, фраза «мой комментарий» помечена как `self` → она отображается со своим стилем + рамкой-исключения.

**Как задать класс заметки:**

а) Через frontmatter:
```yaml
---
yule_class: other
---
```
Допустимые значения: `self`, `ai`, `other`, `none`.

б) Через теги (автоматически, без диалога) — см. ниже.

### 3. Четыре состояния текста

| Состояние | Описание |
|-----------|----------|
| `self`    | Мой текст (явно помечен) |
| `ai`      | Текст ИИ |
| `other`   | Чужой текст |
| *(не помечен)* | Без оформления — текст, который ещё не размечен |

По умолчанию новый текст **не получает авторства** — он «не помечен». Авторство присваивается только явно (вставка с диалогом / команда / тег / frontmatter).

### 4. `yule_class: none` — «не применять»

Для заметок, авторство которых очевидно, можно отключить разметку полностью:
```yaml
---
yule_class: none
---
```
При этом:
- Никакой подсветки не будет, даже если включена глобально.
- Вставка текста не вызывает диалог.
- Команды «Пометить как...» не работают (показывают уведомление).

### 5. Теги без диалога — три отдельных списка

В настройках три независимых списка тегов:
- **Теги → Мой текст** (например: `черновик, личное`)
- **Теги → ИИ** (например: `ai, gpt, chatgpt`)
- **Теги → Чужой текст** (например: `цитации, выписки, вырезки`)

При добавлении тега из списка `noteClass` заметки меняется **автоматически, без диалога**. Приоритет frontmatter выше тегов.

### 6. Команды

| Команда | Описание |
|---------|----------|
| `yule-auth: Пометить как: Мой текст` | Пометить выделение как мой текст |
| `yule-auth: Пометить как: ИИ` | Пометить выделение как ИИ |
| `yule-auth: Пометить как: Чужой текст` | Пометить выделение как чужой текст |
| `yule-auth: Снять авторство (выделение или текущий фрагмент)` | Удалить авторство из БД для выделения или фрагмента под курсором |

**Команда «Toggle Authorship» удалена.** Включение/отключение подсветки — только через настройки.

## Структура базы данных

`authorship-db.json` в папке плагина:

```json
{
  "notes": {
    "path/to/note.md": {
      "noteClass": "other",
      "fragments": [
        {
          "id": "abc123",
          "source": "self",
          "author": "Я",
          "savedFrom": 412,
          "savedTo": 445,
          "startAnchor": "мой комментарий к тексту выше",
          "endAnchor": "мой комментарий к тексту выше",
          "isException": true,
          "created": 1700000000000
        }
      ]
    }
  }
}
```

- `noteClass` — авторство всей заметки (`self`/`ai`/`other`/`none`/`null`).
- `fragments` — явно размеченные фрагменты.
- `isException: true` — фрагмент является исключением внутри noteClass-заметки.

## Установка

1. Скопировать `main.js`, `styles.css`, `manifest.json` в `.obsidian/plugins/yule-auth/`.
2. Включить плагин в настройках Obsidian → Сторонние плагины.

---

## Настройки

- **Подсветка авторства** — вкл/выкл (без команды в палитре).
- **Ваше имя** — как ваши тексты помечаются в БД.
- **Диалог при вставке** — спрашивать авторство при вставке длинных фрагментов.
- **Теги** — три списка тегов для автоматической пометки заметки.
- **Показывать: мой текст / ИИ / чужой текст** — раздельное включение подсветки.
- Настройки фона, цвета шрифта, курсива для каждого из трёх типов.
